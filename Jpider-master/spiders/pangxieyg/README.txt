---------------------------2017.06.06 更新 -----------------------------------------------------------------------
1. 休眠时间可配置，在info.conf里的[main]的sleep中配置，一定要整数，该配置是设置程序发送请求的间隔时间
2. 添加请求超时配置，在[main]的timeout中配置，浮点型，别配置的太短，螃蟹云购响应没那么快，配30s，如果经常报错
   ReadTimeout: HTTPSConnectionPool的错误，可以将timeout时间配置长一点

-------------------------------------------------------------------------------------------------------------------
1. conf文件夹下的info.conf:
	1.1 username为用户名

	1.2 password为密码

	1.3 goods是要监控的商品id，空格分割。

	    商品id可以通过进入商品详细页的url中获取，
		如：‘https://www.pangxieyg.com/wap/tmpl/product_detail.html?goods_id=105289&goods_promotion_type=5’
		是  手机话费充值 3-5个工作日发货 500元 三网通话费充值 商品的url，商品id为105289（goods_id=105289）

	1.4 to是配置下单成功后，邮件接收的邮箱账号，最好不要用QQ邮箱（推荐163），多个账号空格分割（邮件发送账号是u9334486fenpo3@163.com）。

	1.5 thread_num为并发线程数，一般配置小于等于要监控的商品id，多了就浪费了


2. logging.conf文件不要动，是日志输出的

3. 程序运行后，会在根目录生成两个文件夹，log和record
	log文件夹下是运行日志文件，不需要关系
	record文件夹下是各商品每日有库存的时间段

4. notify.wav是下单后的提示音，可以替换，播放时长不要超过10s，必须wav格式

程序只会下单一次，如果想重复下单，可以多开几个程序（放到不同的目录运行）

由于网站不稳定，会报
	ConnectionError: HTTPSConnectionPool(host='www.pangxieyg.com', port=443): Max retries exceeded with url
	ValueError: No JSON object could be decoded
这两个错误，可以忽略，会重试。
